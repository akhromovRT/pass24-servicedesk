# Логирование

Использовать, если скрипт делает сетевые запросы, работает с внешними сервисами или есть риск ошибок.

## Правила
- Использовать минимально необходимый объём данных.
- Создавать один лог на один запуск приложения.
- Путь к логу: папка `logs/run-%timestamp%`.
- В этой папке создать файл `_log.md` и вести все записи в нём.
- Использовать Markdown-разметку.
- Одна запись = одно событие; указывать время и уровень (INFO/WARN/ERROR).
- Удалять логи старше трёх дней, если нет иных инструкций.
- Текст запросов скрывать под `<details>` для экономии места.

## Для backend (FastAPI)
- В production использовать стандартный `logging` Python с JSON-форматом.
- Markdown-логи — только для скриптов и разовых задач агента.

## Минимальный шаблон лога
```markdown
## События
- [HH:MM:SS] [INFO] start - ok
- [HH:MM:SS] [INFO] request <service> <method> <endpoint> - ok
- [HH:MM:SS] [ERROR] <action> - <error>
- [HH:MM:SS] [INFO] finish - ok
```
