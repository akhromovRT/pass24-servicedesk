# Инструкция: создание и ведение проекта внедрения

Руководство для менеджера PASS24 по работе с модулем «Проекты внедрения» на портале поддержки.

---

## 1. Что такое проект внедрения

Проект внедрения — это управление полным циклом установки PASS24 на объекте клиента: от подписания договора до сдачи работ. Проект состоит из **фаз** (этапов), каждая фаза содержит **задачи**. Клиент (администратор УК) видит прогресс своего проекта на портале.

### Типы проектов (шаблоны)

| Тип | Описание | Фаз | Длительность |
|-----|----------|-----|-------------|
| **Стандартный ЖК** | Типовой жилой комплекс: КПП, подъезды, парковка | 10 | ~10 недель |
| **Стандартный БЦ** | Бизнес-центр: турникеты, шлагбаумы, интеграция с 1С/AD | 9 | ~8 недель |
| **Только камеры** | Минимальная установка: распознавание номеров (pass24.auto) | 5 | ~4 недели |
| **Большая стройка** | Масштабный объект: тендер, поэтапная сдача, контроль качества | 12 | ~16 недель |

### Статусы проекта

```
Черновик → Планирование → В работе ⇄ На паузе → Завершён
              ↓                ↓          ↓
           Отменён          Отменён    Отменён
```

| Статус | Когда |
|--------|-------|
| **Черновик** | Проект только создан, заполняются данные |
| **Планирование** | Назначена команда, согласуются сроки |
| **В работе** | Идёт внедрение, задачи выполняются |
| **На паузе** | Ожидаем клиента / решение / поставку |
| **Завершён** | Акт подписан, объект сдан |
| **Отменён** | Договор расторгнут (soft-delete, данные сохраняются) |

## 2. Создание проекта — пошагово

### Шаг 1. Перейти в раздел «Проекты»

В верхнем меню нажмите **«Проекты»** → кнопка **«Создать проект»** (доступна только администраторам).

### Шаг 2. Выбрать или создать клиента

**Если клиент уже есть в системе:**
- В поле «Клиент (администратор УК)» начните вводить ФИО или email — система покажет совпадения
- Выберите нужного из списка

**Если клиента нет в системе (новый клиент):**
1. Нажмите кнопку **«+»** справа от поля выбора клиента
2. В открывшемся окне заполните:
   - **ФИО** — полное имя представителя УК
   - **Email** — на этот адрес придёт welcome-письмо
   - **Телефон** — необязательно
   - **Название УК / компании** — автоматически подставится в поле «Компания»
3. Нажмите **«Создать клиента»**
4. Система покажет **временный пароль** — запишите его или сообщите клиенту устно
5. Нажмите **«Готово»** — клиент автоматически выбран в форме

> **Важно:** клиенту на email автоматически придёт welcome-письмо при создании проекта (не при создании учётной записи). Письмо содержит: ссылку на портал, логин/пароль, список этапов внедрения.

### Шаг 3. Заполнить данные проекта

| Поле | Обязательно | Пример |
|------|-------------|--------|
| **Название проекта** | Да | Внедрение PASS24 в ЖК Зелёный |
| **Компания клиента** | Да | УК Зелёный Двор |
| **Название объекта** | Да | ЖК Зелёный |
| **Адрес объекта** | Нет | Москва, ул. Ленина, 42 |
| **Тип проекта** | Да | Стандартный ЖК |
| **Номер договора** | Нет | ДГ-2026-045 |
| **Дата подписания** | Нет | 05.04.2026 |
| **Плановая дата старта** | Нет | 15.04.2026 |
| **Менеджер проекта** | Нет | Выберите из списка сотрудников PASS24 |
| **Примечания** | Нет | Особые условия, контакты и т.п. |

> **Совет:** если указать «Плановую дату старта» — система автоматически рассчитает плановые даты каждой фазы на основе длительности из шаблона.

### Шаг 4. Проверить предпросмотр шаблона

Справа от формы отображается **предпросмотр выбранного шаблона**: количество фаз, задач, общая длительность и список этапов. Убедитесь, что тип проекта соответствует объекту.

### Шаг 5. Создать проект

Нажмите **«Создать проект»**. Система:
1. Создаст проект с уникальным кодом (PRJ-2026-001)
2. Скопирует все фазы и задачи из шаблона
3. Рассчитает плановые даты (если указан старт)
4. Отправит уведомление клиенту (welcome-email или обычное уведомление)
5. Откроет страницу проекта

## 3. Управление проектом

### Страница проекта

Страница проекта содержит **7 вкладок**:

| Вкладка | Что содержит |
|---------|-------------|
| **Этапы** | Раскрывающиеся карточки фаз с задачами |
| **Timeline** | Вертикальная визуализация фаз с маркерами статуса |
| **Документы** | Загрузка и скачивание файлов (договоры, акты, фото) |
| **Команда** | Участники проекта с ролями |
| **Тикеты** | Связанные заявки (в т.ч. блокеры) |
| **Комментарии** | Обсуждение (публичные + внутренние) |
| **История** | Лента всех событий проекта |

### Смена статуса проекта

На странице проекта вверху — кнопки доступных переходов:

- **Черновик** → «В планирование» или «Отменить»
- **Планирование** → «Запустить» (переводит в «В работе», фиксирует дату старта) или «Отменить»
- **В работе** → «На паузу» или «Завершить»
- **На паузе** → «Продолжить» или «Отменить»

### Работа с фазами

Каждая фаза — раскрывающаяся карточка:

1. **Старт фазы** — нажмите кнопку «Старт» на карточке. Фиксируется фактическая дата начала.
2. **Изменение дат** — кликните на даты (пунктирная линия) → откроется inline-календарь. Измените и нажмите ✓.
3. **Завершение фазы** — нажмите «Завершить». Прогресс фазы ставится 100%, обновляется общий прогресс проекта. Клиент получит email-уведомление.

### Работа с задачами

Задачи внутри фазы:
- **Выполнить** — нажмите ✓ на строке задачи. Прогресс фазы и проекта пересчитается автоматически.
- **Отменить** — нажмите ✕ (задача помечается как отменённая, не учитывается в прогрессе).
- **Добавить новую** — нажмите «Добавить задачу» внизу раскрытой фазы. Укажите название и отметьте «Milestone» если это ключевая веха (при выполнении milestone-задачи клиент получит email).

### Прогресс

Прогресс рассчитывается автоматически:
- **Фаза**: `выполненные задачи / активные задачи × 100%` (отменённые не считаются)
- **Проект**: средневзвешенное по фазам с учётом веса (пропущенные фазы не учитываются)

### Документы

Во вкладке «Документы»:
1. Выберите **тип документа** (Договор / ТЗ / Акт / Схема / Фото / Отчёт / Другое)
2. Нажмите **«Загрузить файл»** и выберите файл (до 20 МБ, PDF/DOCX/XLSX/JPG/PNG/TXT)
3. Для скачивания — нажмите значок ⬇ на строке документа
4. Для удаления — нажмите значок 🗑 (только для сотрудников PASS24)

> **Клиенту** доступна загрузка документов в свой проект (но не удаление).

### Связь с тикетами

Если в процессе внедрения возникает проблема — её можно оформить как тикет и привязать к проекту:
1. На странице тикета (раздел «Заявки») в блоке **«Проект внедрения»** выберите проект из списка
2. Отметьте **«Блокирует проект»** если тикет критичен для продолжения внедрения
3. На странице проекта (вкладка «Тикеты») будут видны все привязанные заявки — блокеры выделены красным

### Комментарии

- **Публичные** — видны клиенту и команде PASS24
- **Внутренние** (галочка «Внутренний (только PASS24)») — видны только команде, клиент их не увидит

### Редактирование проекта

Нажмите **«Редактировать»** вверху страницы → откроется окно с полями:
- Название, компания, объект, адрес
- Плановые даты старта и завершения
- Примечания

## 4. Что видит клиент

Администратор УК (property_manager) видит на портале:

- **Раздел «Проекты»** в меню — только свои проекты
- **Страница проекта**: все вкладки, прогресс, timeline, документы
- **Комментарии** — только публичные (внутренние скрыты)
- **Загрузка документов** — может загружать файлы, но не удалять
- **Заявки** — может видеть связанные тикеты и создавать новые

Клиент **не может**:
- Создавать / редактировать / удалять проекты
- Менять статусы фаз и задач
- Удалять документы
- Писать внутренние комментарии
- Управлять командой

---

## 5. Email-уведомления

Клиент получает письма автоматически:

| Событие | Что в письме |
|---------|-------------|
| **Создание проекта (новый клиент)** | Welcome: ссылка на портал, логин/пароль, возможности портала, список этапов с длительностями |
| **Создание проекта (существующий)** | Код проекта, название, объект, нумерованный список этапов |
| **Изменение статуса** | Какой статус, кто изменил |
| **Завершение фазы** | Название фазы, текущий % прогресса |
| **Milestone-задача выполнена** | Название вехи, кто выполнил |

## 6. Типичные сценарии

### Новый клиент, стандартный ЖК

1. «Проекты» → «Создать проект»
2. Кнопка «+» → создать клиента (ФИО, email, компания)
3. Записать временный пароль
4. Заполнить: название, объект, тип «Стандартный ЖК», дату старта
5. «Создать проект» → клиенту уходит welcome-email
6. Перевести проект: Черновик → Планирование → В работе
7. По мере выполнения — завершать задачи, стартовать/завершать фазы
8. По завершении — статус «Завершён»

### Только установка камер

1. «Создать проект» → тип «Только камеры» (5 фаз, ~4 недели)
2. Выбрать существующего клиента
3. После создания — сразу перевести в «В работе»
4. Укороченный цикл: Planning → Installation → Configuration → Testing → Go-Live

### Проблема при внедрении → тикет-блокер

1. Клиент или менеджер создаёт тикет в разделе «Заявки»
2. На странице тикета → «Проект внедрения» → выбрать проект → ✓ «Блокирует проект»
3. На странице проекта (вкладка «Тикеты») тикет виден с красной меткой «Блокер»
4. После решения тикета — снять связь или убрать флаг блокера
