# Регламент работы PASS24 Service Desk

Инструкция для менеджера по поддержке. Описывает все процессы обработки обращений клиентов в системе PASS24 Service Desk.

---

## 1. Общие сведения

### Что такое PASS24 Service Desk

Веб-портал технической поддержки для пользователей СКУД-системы PASS24.online. Через портал клиенты (жители ЖК, сотрудники БЦ, администраторы управляющих компаний) обращаются с проблемами, а команда поддержки обрабатывает заявки и наполняет базу знаний.

### Каналы поступления заявок

| Канал | Как работает |
|-------|-------------|
| **Веб-портал** | Клиент заходит на сайт, авторизуется и создаёт заявку через форму |
| **Email** | Клиент пишет на `support@pass24online.ru` — заявка создаётся автоматически |

### Роли в системе

| Роль | Кто | Возможности |
|------|-----|-------------|
| **Житель (resident)** | Жители ЖК, сотрудники БЦ | Создание заявок, просмотр своих заявок, чтение базы знаний |
| **Администратор УК (property_manager)** | Управляющие компании | То же + просмотр заявок по своему объекту |
| **Агент поддержки (support_agent)** | Команда PASS24 | Обработка всех заявок, управление базой знаний |
| **Администратор (admin)** | Руководитель поддержки | Полный доступ: удаление заявок, управление пользователями |

## 2. Вход в систему

### Первичная регистрация

1. Откройте Service Desk в браузере
2. Нажмите **«Регистрация»** на странице входа
3. Заполните форму:
   - **Имя** — ваше полное имя (будет видно клиентам в комментариях и уведомлениях)
   - **Email** — рабочий адрес электронной почты
   - **Пароль** — минимум 6 символов
   - **Роль** — выберите «Агент поддержки»
4. Нажмите **«Зарегистрироваться»**
5. Система перенаправит вас на главную страницу — список заявок

### Ежедневный вход

1. Откройте Service Desk
2. Введите email и пароль
3. Нажмите **«Войти»**

> **Важно:** Сессия действует 24 часа. После этого потребуется повторный вход.

## 3. Работа с заявками

### 3.1. Просмотр списка заявок

Главная страница — таблица всех заявок. Отображаются колонки:
- **Тема** — краткое описание проблемы
- **Категория** — тип проблемы (Доступ, Шлагбаум, Пропуска и т.д.)
- **Статус** — текущий этап обработки (цветная метка)
- **Приоритет** — важность заявки (цветная метка)
- **Дата создания** — когда заявка поступила

#### Фильтрация

Используйте фильтры в верхней панели:
- **По статусу:** Все / Новые / В работе / Ожидает ответа / Решённые / Закрытые
- **По категории:** Все / Доступ / Пропуска / Шлагбаум / Уведомления / Общее

> **Рекомендация:** В начале рабочего дня фильтруйте по статусу «Новые» чтобы увидеть необработанные заявки.

### 3.2. Приоритеты заявок

Система автоматически назначает приоритет при создании заявки на основе ключевых слов:

| Приоритет | Цвет | Когда назначается | Время реакции |
|-----------|------|-------------------|---------------|
| **Критический** | Красный | «Не могу попасть», «дверь не открылась», «не пускает» | Немедленно |
| **Высокий** | Оранжевый | «Шлагбаум», «парковка» | В течение 1 часа |
| **Обычный** | Синий | Все остальные обращения | В течение 4 часов |
| **Низкий** | Серый | «Уведомления», «пуш» | В течение 8 часов |

> **Правило:** Критические и высокие заявки обрабатываются в первую очередь. Если видите красную метку — реагируйте немедленно.

### 3.3. Жизненный цикл заявки (статусы)

```
  НОВЫЙ ──────────────────────┐
    │                         │
    ▼                         ▼
  В РАБОТЕ              РЕШЁН (быстрое решение)
    │                    │         │
    ├──► ОЖИДАЕТ ОТВЕТА  │         │
    │         │          │         │
    │         └──► В РАБОТЕ        │
    │                    │         │
    └──► РЕШЁН ◄─────────┘         │
           │                       │
           ├──► ЗАКРЫТ ◄───────────┘
           │
           └──► В РАБОТЕ (переоткрытие)
```

| Статус | Значение | Что делать |
|--------|----------|-----------|
| **Новый** | Заявка только поступила | Взять в работу или сразу решить |
| **В работе** | Агент занимается проблемой | Разбираться, комментировать ход работ |
| **Ожидает ответа** | Нужна информация от клиента | Ждать ответа клиента, через 48ч — напомнить |
| **Решён** | Проблема устранена | Ждать подтверждения от клиента, затем закрыть |
| **Закрыт** | Заявка завершена | Финальный статус, изменить нельзя |

### 3.4. Обработка новой заявки

**Шаг 1. Открыть заявку**
- Нажмите на строку в таблице → откроется страница заявки

**Шаг 2. Оценить проблему**
- Прочитайте тему и описание
- Проверьте категорию и приоритет (при необходимости скорректируйте через комментарий)
- Определите, можете ли решить проблему самостоятельно

**Шаг 3. Взять в работу**
- Нажмите кнопку **«Взять в работу»** → статус изменится на «В работе»
- Клиент получит email-уведомление о смене статуса

**Шаг 4. Работа над заявкой**

Вариант A — **Проблема понятна, решение известно:**
1. Выполните необходимые действия (проверка в системе PASS24, настройка доступа и т.д.)
2. Напишите комментарий с описанием что было сделано
3. Нажмите **«Решить»** → статус «Решён»

Вариант B — **Нужна дополнительная информация от клиента:**
1. Напишите комментарий с вопросами (адрес объекта, номер пропуска, скриншот и т.д.)
2. Нажмите **«Ожидать ответа»** → статус «Ожидает ответа»
3. Клиент получит email с вашим вопросом

Вариант C — **Быстрое решение:**
- Если заявка типовая и решается мгновенно (например, сброс QR-кода), можно из статуса «Новый» сразу нажать **«Решить»**

**Шаг 5. Закрытие**
- После решения: если клиент подтвердил или не отвечает 3 дня → нажмите **«Закрыть»**
- Закрытый тикет больше нельзя изменить

### 3.5. Комментирование

Комментарии — основной инструмент коммуникации с клиентом.

**Как добавить комментарий:**
1. Откройте заявку
2. Внизу страницы найдите поле «Добавить комментарий»
3. Напишите текст
4. Нажмите **«Отправить»**

**Клиент получит email-уведомление** с текстом вашего комментария.

**Правила комментирования:**
- Пишите понятным языком, без технического жаргона
- Указывайте конкретные действия: «Мы перевыпустили ваш QR-код, попробуйте ещё раз»
- Если нужна информация — задавайте чёткие вопросы: «Укажите номер подъезда и этаж»
- Не оставляйте заявку без комментария более 4 часов (для обычного приоритета)

### 3.6. Переоткрытие заявки

Если клиент сообщает, что проблема не решена:
1. Откройте решённую заявку
2. Нажмите **«Переоткрыть»** → статус вернётся на «В работе»
3. Добавьте комментарий с описанием проблемы
4. Повторите процесс решения

## 4. Обработка email-заявок

### Как это работает

Система автоматически проверяет почту `support@pass24online.ru` каждые 60 секунд.

**Сценарий 1 — Полноценное письмо:**
- Клиент пишет на `support@pass24online.ru` с темой и подробным описанием
- Система автоматически создаёт тикет:
  - Тема письма → название заявки
  - Тело письма → описание
  - Определяет категорию по ключевым словам
  - Назначает приоритет автоматически
- Клиенту приходит ответ: «Заявка принята» с номером и приоритетом
- Если отправитель ранее не регистрировался — система создаёт ему учётную запись

**Сценарий 2 — Недостаточно информации:**
- Клиент пишет коротко: «Помогите» или «Не работает»
- Система отвечает письмом с запросом уточнений:
  - Тема обращения
  - Подробное описание проблемы
  - Адрес объекта (ЖК/БЦ, подъезд)
  - Контактный телефон
- Тикет НЕ создаётся до получения полной информации

### Что делать менеджеру

- Email-заявки появляются в общем списке наравне с заявками из веб-портала
- В описании будет пометка «Тикет создан из email»
- Обрабатывайте их по стандартной процедуре (раздел 3.4)
- Все ваши комментарии будут отправлены клиенту на email

## 5. База знаний

### Зачем нужна

База знаний — это FAQ и инструкции для клиентов. Чем больше статей — тем меньше типовых заявок. Клиенты могут найти ответ самостоятельно, не создавая тикет.

### Создание статьи

1. Перейдите в раздел **«База знаний»** в меню
2. Нажмите кнопку **«Создать статью»** (видна только агентам и администраторам)
3. Заполните:
   - **Заголовок** — чёткий вопрос или название инструкции
     - Хорошо: «Как обновить QR-код пропуска в приложении PASS24»
     - Плохо: «Пропуска»
   - **Категория** — выберите из списка (Доступ, Пропуска, Шлагбаумы, Приложение, Уведомления, Общее)
   - **Содержание** — подробная инструкция в формате Markdown
4. Нажмите **«Сохранить»**

### Категории статей

| Категория | Примеры тем |
|-----------|-------------|
| **Доступ** | Дверь не открывается, домофон не работает, проблемы со входом |
| **Пропуска** | QR-код, выпуск гостевого пропуска, продление пропуска |
| **Шлагбаумы** | Шлагбаум не открывается, парковка, въезд/выезд |
| **Приложение** | Установка, обновление, вход в мобильное приложение |
| **Уведомления** | Push-уведомления, SMS, email-оповещения |
| **Общее** | Регистрация, оплата, договор, прочие вопросы |

### Формат содержания (Markdown)

```markdown
# Заголовок инструкции

Вводное предложение — когда и кому пригодится эта инструкция.

## Пошаговая инструкция

1. Откройте приложение PASS24
2. Перейдите в раздел «Мои пропуска»
3. Нажмите кнопку «Обновить QR-код»
4. Дождитесь уведомления «QR-код обновлён»

## Если не помогло

Обратитесь в поддержку:
- Через портал Service Desk
- По email: support@pass24online.ru
```

### Когда создавать статью

- Получили **3+ одинаковых заявки** на одну тему → создайте статью
- **Вышло обновление** PASS24 → обновите или создайте инструкцию
- Клиент прислал **нестандартный вопрос** с полезным решением → зафиксируйте

## 7. Ежедневный порядок работы

### Утро (начало смены)

1. Войдите в Service Desk
2. Отфильтруйте заявки по статусу **«Новые»**
3. Проверьте **критические** заявки (красная метка) — обработайте их первыми
4. Возьмите в работу **высокие** заявки (оранжевая метка)
5. Проверьте заявки в статусе **«Ожидает ответа»** — если ответ пришёл, верните в работу

### В течение дня

- Обрабатывайте новые заявки по мере поступления
- Отвечайте на комментарии клиентов
- Проверяйте email на `support@pass24online.ru` — убедитесь, что заявки создаются автоматически

### Вечер (конец смены)

1. Проверьте все свои заявки в статусе «В работе»
2. Если не можете завершить сегодня — напишите комментарий о статусе работ
3. Закройте решённые заявки, по которым клиент подтвердил решение

### Еженедельно

- Проверьте заявки в статусе **«Ожидает ответа»** более 48 часов — напомните клиенту
- Проверьте решённые заявки старше 3 дней — закройте их
- Создайте статьи в базе знаний по типовым вопросам недели

### При закрытии без ответа
> Заявка закрыта в связи с отсутствием ответа. Если проблема актуальна — создайте новое обращение.

> По вашему вопросу у нас есть готовая инструкция: [ссылка на статью в базе знаний]. Если она не поможет — напишите, разберёмся детальнее.

## 9. Типовые ситуации и решения

### «Не могу попасть в подъезд» (CRITICAL)

1. Уточните адрес, номер подъезда, номер квартиры
2. Проверьте в системе PASS24: активен ли пропуск, привязан ли к объекту
3. Если пропуск неактивен → активируйте, уведомите клиента
4. Если проблема с оборудованием → передайте в отдел обслуживания, уведомите клиента о сроках

### «Шлагбаум не открывается» (HIGH)

1. Уточните адрес паркинга и номер автомобиля
2. Проверьте привязку автомобиля к пропуску в PASS24
3. Если проблема с распознаванием номера → предложите ручное открытие через приложение
4. Если техническая неисправность → сообщите время ожидания ремонта

### «Не приходят уведомления» (LOW)

1. Уточните тип устройства (Android/iOS)
2. Предложите проверить: разрешения приложения → уведомления → включить
3. Направьте на статью в базе знаний
4. Если не помогает → попросите переустановить приложение

### Повторное обращение по решённой заявке

1. Клиент отвечает на email-уведомление → новое письмо попадает в систему
2. Найдите исходный тикет по теме
3. Если заявка ещё не закрыта — переоткройте
4. Если закрыта — создайте новый тикет со ссылкой на предыдущий
