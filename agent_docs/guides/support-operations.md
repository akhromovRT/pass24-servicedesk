# Регламент работы PASS24 Service Desk

Инструкция для менеджера по поддержке. Описывает все процессы обработки обращений клиентов в системе PASS24 Service Desk.

---

## 1. Общие сведения

### Что такое PASS24 Service Desk

Веб-портал технической поддержки для пользователей СКУД-системы PASS24.online. Через портал клиенты (жители ЖК, сотрудники БЦ, администраторы управляющих компаний) обращаются с проблемами, а команда поддержки обрабатывает заявки и наполняет базу знаний.

### Каналы поступления заявок

| Канал | Как работает |
|-------|-------------|
| **Веб-портал** | Клиент заходит на сайт, авторизуется и создаёт заявку через форму |
| **Email** | Клиент пишет на `support@pass24online.ru` — заявка создаётся автоматически |

### Роли в системе

| Роль | Кто | Возможности |
|------|-----|-------------|
| **Житель (resident)** | Жители ЖК, сотрудники БЦ | Создание заявок, просмотр своих заявок, чтение базы знаний |
| **Администратор УК (property_manager)** | Управляющие компании | То же + просмотр заявок по своему объекту |
| **Агент поддержки (support_agent)** | Команда PASS24 | Обработка всех заявок, управление базой знаний |
| **Администратор (admin)** | Руководитель поддержки | Полный доступ: удаление заявок, управление пользователями |

## 2. Вход в систему

### Первичная регистрация

1. Откройте Service Desk в браузере
2. Нажмите **«Регистрация»** на странице входа
3. Заполните форму:
   - **Имя** — ваше полное имя (будет видно клиентам в сообщениях и уведомлениях)
   - **Email** — рабочий адрес электронной почты
   - **Пароль** — минимум 6 символов
   - **Роль** — выберите «Агент поддержки»
4. Нажмите **«Зарегистрироваться»**
5. Система перенаправит вас на главную страницу — список заявок

### Ежедневный вход

1. Откройте Service Desk
2. Введите email и пароль
3. Нажмите **«Войти»**

> **Важно:** Сессия действует 24 часа. После этого потребуется повторный вход.

## 3. Работа с заявками

### 3.1. Просмотр списка заявок

Главная страница — таблица всех заявок. Отображаются колонки:
- **Тема** — краткое описание проблемы
- **Категория** — тип проблемы (Доступ, Шлагбаум, Пропуска и т.д.)
- **Статус** — текущий этап обработки (цветная метка)
- **Приоритет** — важность заявки (цветная метка)
- **Дата создания** — когда заявка поступила

#### Вкладки

Над таблицей расположены вкладки для быстрого доступа к нужным заявкам:
- **Все** — полный список заявок
- **Открытые** — заявки, требующие внимания (не закрытые и не решённые)
- **Срочные** — заявки с высоким и критическим приоритетом
- **Просрочено** — заявки, по которым истёк срок реакции
- **Ждут ответа** — заявки в статусе «Ожидает ответа»
- **Выезды** — заявки в статусе «Выезд инженера» (инженер выехал на объект)
- **Закрытые** — завершённые заявки

#### Сортировка

По умолчанию заявки отсортированы по дате — **«Сначала новые»**. Самые свежие заявки отображаются вверху списка. Вы можете изменить сортировку через выпадающий список в верхней панели.

#### Фильтрация

Используйте фильтры в верхней панели:
- **По статусу:** Все / Новые / В работе / Ожидает ответа / Отложена / Выезд инженера / Решённые / Закрытые
- **По категории:** Все / Доступ / Пропуска / Шлагбаум / Уведомления / Общее
- **«Постоянные клиенты / Все клиенты»** (тогл-кнопка с золотой звездой) — переключает ленту на заявки только от компаний, помеченных в Bitrix24 как «Постоянный клиент». Подробнее — `agent_docs/guides/permanent-clients.md`.

> **Рекомендация:** В начале рабочего дня откройте вкладку **«Открытые»** и отсортируйте по дате, чтобы увидеть последние поступившие заявки.

#### Маркеры в строке заявки

В мета-строке карточки (под заголовком) могут отображаться:
- ⭐ **Постоянный** (жёлтый) — заявка от компании из справочника постоянных клиентов Bitrix24. Бейдж виден только агентам и админам. Используйте, чтобы быстрее замечать заявки от ключевых клиентов.
- **Отв.: клиент / Отв.: оператор** — кто ответил последним в публичной переписке.
- ⬤ — синяя точка слева от заголовка: новый, ещё не просмотренный ответ клиента.

### 3.2. Приоритеты заявок

Система автоматически назначает приоритет при создании заявки на основе ключевых слов:

| Приоритет | Цвет | Когда назначается | Время реакции |
|-----------|------|-------------------|---------------|
| **Критический** | Красный | «Не могу попасть», «дверь не открылась», «не пускает» | Немедленно |
| **Высокий** | Оранжевый | «Шлагбаум», «парковка» | В течение 1 часа |
| **Обычный** | Синий | Все остальные обращения | В течение 4 часов |
| **Низкий** | Серый | «Уведомления», «пуш» | В течение 8 часов |

> **Правило:** Критические и высокие заявки обрабатываются в первую очередь. Если видите красную метку — реагируйте немедленно.

### 3.3. Жизненный цикл заявки (статусы)

Заявка проходит через 7 статусов. Ниже — схема всех возможных переходов:

```
  НОВЫЙ ─────────────────────────────────┐
    │                                    │
    ▼                                    ▼
  В РАБОТЕ                          РЕШЁН (быстрое решение)
    │                                │         │
    ├──► ОЖИДАЕТ ОТВЕТА ──► В РАБОТЕ │         │
    │                                │         │
    ├──► ОТЛОЖЕНА ──────────► В РАБОТЕ         │
    │                                │         │
    ├──► ВЫЕЗД ИНЖЕНЕРА             │         │
    │       │                        │         │
    │       ├──► В РАБОТЕ            │         │
    │       ├──► ОЖИДАЕТ ОТВЕТА      │         │
    │       └──► РЕШЁН               │         │
    │                                │         │
    └──► РЕШЁН ◄─────────────────────┘         │
           │                                   │
           ├──► ЗАКРЫТ ◄───────────────────────┘
           │
           └──► В РАБОТЕ (переоткрытие)
```

| Статус | Значение | Что делать |
|--------|----------|-----------|
| **Новый** | Заявка только поступила | Взять в работу или сразу решить |
| **В работе** | Агент занимается проблемой | Разбираться, писать клиенту, менять статус по ситуации |
| **Ожидает ответа** | Нужна информация от клиента | Ждать ответа клиента, через 48 ч — напомнить |
| **Отложена** | Заявка приостановлена (ждём оборудование, подрядчика и т.д.) | Таймер срока реакции останавливается. Вернуть в работу, когда причина ожидания устранена |
| **Выезд инженера** | Инженер выехал на объект | Таймер срока реакции продолжает идти — работа ведётся. Обновить статус по результатам визита |
| **Решён** | Проблема устранена | Ждать подтверждения от клиента, затем закрыть |
| **Закрыт** | Заявка завершена | Финальный статус, изменить нельзя |

#### Как менять статус

Статус заявки меняется через **выпадающий список** в правой панели карточки заявки:

1. Откройте заявку
2. В правой панели найдите текущий статус (цветная метка, например «Новый»)
3. Нажмите на статус — откроется выпадающий список доступных действий
4. Выберите нужное действие

**Какие действия доступны из каждого статуса:**

| Текущий статус | Доступные действия в выпадающем списке |
|----------------|---------------------------------------|
| **Новый** | **«Взять в работу»**, **«Решить»** |
| **В работе** | **«Ожидать ответа»**, **«Отложить»**, **«Выезд инженера»**, **«Решить»** |
| **Ожидает ответа** | **«Вернуть в работу»**, **«Решить»** |
| **Отложена** | **«Вернуть в работу»**, **«Решить»** |
| **Выезд инженера** | **«Вернуть в работу»**, **«Ожидать ответа»**, **«Решить»** |
| **Решён** | **«Закрыть»**, **«Переоткрыть»** |
| **Закрыт** | Нет доступных действий — финальный статус |

### 3.4. Обработка новой заявки

При открытии заявки вы увидите страницу с двумя колонками:
- **Левая часть (основная)** — переписка с клиентом в виде чата
- **Правая панель** — информация о заявке и управление

#### Правая панель содержит:

- **Статус** — выпадающий список для смены статуса (см. раздел 3.3)
- **Срок реакции** — полоса прогресса, показывающая сколько времени осталось до истечения срока
- **Назначение** — кому назначена заявка
- **Контактная информация** — данные клиента (имя, email, телефон)
- **Объект** — адрес ЖК/БЦ, к которому относится заявка
- **Классификация** — категория и приоритет заявки
- **Макросы** — готовые шаблоны ответов для быстрой работы
- **История событий** — лог всех действий по заявке (смена статуса, назначение и т.д.)

#### Порядок обработки

**Шаг 1. Открыть заявку**
- Нажмите на строку в таблице → откроется карточка заявки с двумя колонками

**Шаг 2. Оценить проблему**
- Прочитайте переписку в левой части — сообщения клиента отображаются слева, ваши ответы — справа
- В правой панели проверьте категорию, приоритет, контактные данные и объект
- Определите, можете ли решить проблему самостоятельно

**Шаг 3. Взять в работу**
- В правой панели нажмите на статус **«Новый»** и выберите **«Взять в работу»** → статус изменится на «В работе»
- Клиент получит email-уведомление о смене статуса

> **Автоматическая смена статуса:** Когда вы пишете первое сообщение клиенту по новой заявке, статус автоматически меняется с «Новый» на «В работе». Это значит, что достаточно просто написать ответ — вручную менять статус на «В работе» не обязательно.

**Шаг 4. Работа над заявкой**

Вариант A — **Проблема понятна, решение известно:**
1. Выполните необходимые действия (проверка в системе PASS24, настройка доступа и т.д.)
2. Напишите клиенту в чате, что было сделано
3. В правой панели нажмите на статус и выберите **«Решить»** → статус станет «Решён»

Вариант B — **Нужна дополнительная информация от клиента:**
1. Напишите клиенту в чате вопросы (адрес объекта, номер пропуска, скриншот и т.д.)
2. Клиент получит email с вашим вопросом
3. SLA «Решение» автоматически встанет на паузу, пока клиент не ответит
4. Статус **«Ожидает ответа»** можно выставить вручную — как явный маркер для коллег, но он больше не обязателен для паузы SLA

Вариант C — **Нужен выезд инженера:**
1. Напишите клиенту, что для решения проблемы требуется выезд на объект
2. В правой панели нажмите на статус и выберите **«Выезд инженера»**
3. Заявка появится во вкладке **«Выезды»** в списке заявок
4. Пока ваше сообщение было последним в переписке — SLA «Решение» автоматически на паузе. Как только клиент ответит — таймер продолжит тикать.

Вариант D — **Заявку нужно приостановить (ждём оборудование, подрядчика и т.д.):**
1. Напишите клиенту причину ожидания
2. В правой панели нажмите на статус и выберите **«Отложить»**
3. Таймер срока реакции остановится на время паузы
4. Когда причина ожидания устранена — выберите **«Вернуть в работу»**

Вариант E — **Быстрое решение:**
- Если заявка типовая и решается мгновенно (например, сброс QR-кода), можно из статуса «Новый» сразу выбрать **«Решить»**

**Шаг 5. Закрытие**
- После решения: если клиент подтвердил или не отвечает 3 дня → в правой панели выберите **«Закрыть»**
- Закрытую заявку больше нельзя изменить

### 3.5. Переписка с клиентом (чат)

Переписка с клиентом отображается в левой части карточки заявки в виде чата:
- **Сообщения клиента** — отображаются слева, как входящие
- **Ваши сообщения** — отображаются справа, как исходящие
- **События** (смена статуса, назначение) — отображаются как разделители между сообщениями

#### Вложения в сообщениях

Вложения отображаются прямо внутри сообщения:
- **Картинки** — показываются как миниатюры, можно нажать для просмотра в полном размере
- **Файлы** — показываются как карточка с именем файла и размером, можно нажать для скачивания

#### Как отправить сообщение

1. Внизу чата найдите поле **«Напишите ответ...»**
2. Введите текст сообщения
3. Если нужно прикрепить файл — нажмите кнопку со значком скрепки (📎) слева от поля ввода
4. Нажмите **«Отправить»**

Клиент получит email-уведомление с текстом вашего сообщения.

#### Внутренние сообщения (заметки для коллег)

Если нужно оставить заметку, которую увидят **только агенты** (клиент её не увидит и не получит уведомление):

1. Перед отправкой поставьте галочку **«Внутренний»** рядом с кнопкой **«Отправить»**
2. Напишите текст и нажмите **«Отправить»**

Внутренние сообщения выделяются **жёлтым фоном** и помечены значком замка, чтобы отличать их от обычных сообщений клиенту.

> **Когда использовать внутренние сообщения:**
> - Передать информацию коллеге, который будет продолжать работу по заявке
> - Зафиксировать внутренний комментарий по ситуации
> - Оставить заметку для себя перед концом смены

#### Правила переписки

- Пишите понятным языком, без технического жаргона
- Указывайте конкретные действия: «Мы перевыпустили ваш QR-код, попробуйте ещё раз»
- Если нужна информация — задавайте чёткие вопросы: «Укажите номер подъезда и этаж»
- Не оставляйте заявку без ответа более 4 часов (для обычного приоритета)

### 3.6. Переоткрытие заявки

Если клиент сообщает, что проблема не решена:
1. Откройте решённую заявку
2. В правой панели нажмите на статус и выберите **«Переоткрыть»** → статус вернётся на «В работе»
3. Напишите в чате комментарий с описанием проблемы
4. Повторите процесс решения

## 4. Обработка email-заявок

### Как это работает

Система автоматически проверяет почту `support@pass24online.ru` каждые 60 секунд.

**Сценарий 1 — Полноценное письмо:**
- Клиент пишет на `support@pass24online.ru` с темой и подробным описанием
- Система автоматически создаёт заявку:
  - Тема письма → название заявки
  - Тело письма → описание
  - Определяет категорию по ключевым словам
  - Назначает приоритет автоматически
- Клиенту приходит ответ: «Заявка принята» с номером и приоритетом
- Если отправитель ранее не регистрировался — система создаёт ему учётную запись

**Сценарий 2 — Недостаточно информации:**
- Клиент пишет коротко: «Помогите» или «Не работает»
- Система отвечает письмом с запросом уточнений:
  - Тема обращения
  - Подробное описание проблемы
  - Адрес объекта (ЖК/БЦ, подъезд)
  - Контактный телефон
- Заявка НЕ создаётся до получения полной информации

### Как работает привязка писем к заявкам

Когда клиент отвечает на уведомление по заявке, его письмо автоматически попадает в правильную заявку. Номер заявки содержится в теле письма, поэтому привязка работает корректно даже если почтовый клиент изменит тему письма.

### Что делать менеджеру

- Email-заявки появляются в общем списке наравне с заявками из веб-портала
- В описании будет пометка «Тикет создан из email»
- Обрабатывайте их по стандартной процедуре (раздел 3.4)
- Все ваши сообщения в чате будут отправлены клиенту на email

## 5. База знаний

### Зачем нужна

База знаний — это FAQ и инструкции для клиентов. Чем больше статей — тем меньше типовых заявок. Клиенты могут найти ответ самостоятельно, не создавая заявку.

### Создание статьи

1. Перейдите в раздел **«База знаний»** в меню
2. Нажмите кнопку **«Создать статью»** (видна только агентам и администраторам)
3. Заполните:
   - **Заголовок** — чёткий вопрос или название инструкции
     - Хорошо: «Как обновить QR-код пропуска в приложении PASS24»
     - Плохо: «Пропуска»
   - **Категория** — выберите из списка (Доступ, Пропуска, Шлагбаумы, Приложение, Уведомления, Общее)
   - **Содержание** — подробная инструкция
4. Нажмите **«Сохранить»**

### Категории статей

| Категория | Примеры тем |
|-----------|-------------|
| **Доступ** | Дверь не открывается, домофон не работает, проблемы со входом |
| **Пропуска** | QR-код, выпуск гостевого пропуска, продление пропуска |
| **Шлагбаумы** | Шлагбаум не открывается, парковка, въезд/выезд |
| **Приложение** | Установка, обновление, вход в мобильное приложение |
| **Уведомления** | Push-уведомления, SMS, email-оповещения |
| **Общее** | Регистрация, оплата, договор, прочие вопросы |

### Формат содержания

```markdown
# Заголовок инструкции

Вводное предложение — когда и кому пригодится эта инструкция.

## Пошаговая инструкция

1. Откройте приложение PASS24
2. Перейдите в раздел «Мои пропуска»
```

## 7. Ежедневный порядок работы

### Утро (начало смены)

1. Войдите в Service Desk
2. Откройте вкладку **«Открытые»** — увидите все заявки, требующие внимания
3. Проверьте **критические** заявки (красная метка) — обработайте их первыми
4. Возьмите в работу **высокие** заявки (оранжевая метка)
5. Проверьте вкладку **«Ждут ответа»** — если ответ от клиента пришёл, верните заявку в работу
6. Проверьте вкладку **«Выезды»** — обновите статус по результатам выездов инженеров

### В течение дня

- Обрабатывайте новые заявки по мере поступления
- Отвечайте на сообщения клиентов в чате
- Проверяйте email на `support@pass24online.ru` — убедитесь, что заявки создаются автоматически
- Следите за полосой срока реакции в правой панели — не допускайте просрочки

### Вечер (конец смены)

1. Проверьте все свои заявки в статусе «В работе»
2. Если не можете завершить сегодня — оставьте внутреннее сообщение (галочка **«Внутренний»**) о статусе работ для коллег
3. Закройте решённые заявки, по которым клиент подтвердил решение

### Еженедельно

- Проверьте заявки в статусе **«Ожидает ответа»** более 48 часов — напомните клиенту
- Проверьте **отложенные** заявки — если причина ожидания устранена, верните в работу
- Проверьте решённые заявки старше 3 дней — закройте их
- Создайте статьи в базе знаний по типовым вопросам недели

## 9. Типовые ситуации и решения

### «Не могу попасть в подъезд» (критический)

1. Уточните адрес, номер подъезда, номер квартиры
2. Проверьте в системе PASS24: активен ли пропуск, привязан ли к объекту
3. Если пропуск неактивен → активируйте, уведомите клиента
4. Если проблема с оборудованием → выберите статус **«Выезд инженера»** или **«Отложить»** (если ждём запчасти), уведомите клиента о сроках

### «Шлагбаум не открывается» (высокий)

1. Уточните адрес паркинга и номер автомобиля
2. Проверьте привязку автомобиля к пропуску в PASS24
3. Если проблема с распознаванием номера → предложите ручное открытие через приложение
4. Если техническая неисправность → установите статус **«Выезд инженера»**, сообщите время ожидания ремонта

### «Не приходят уведомления» (низкий)

1. Уточните тип устройства (Android/iOS)
2. Предложите проверить: разрешения приложения → уведомления → включить
3. Направьте на статью в базе знаний
4. Если не помогает → попросите переустановить приложение

### Повторное обращение по решённой заявке

1. Клиент отвечает на email-уведомление → письмо автоматически попадает в нужную заявку
2. Если заявка ещё не закрыта — выберите **«Переоткрыть»**
3. Если закрыта — создайте новую заявку со ссылкой на предыдущую
