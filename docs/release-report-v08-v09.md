# Отчёт о реализации v0.8 и v0.9 — PASS24 Service Desk

**Период:** 7-9 апреля 2026
**Статус:** Развёрнуто на production (support.pass24pro.ru)

---

## 1. Редизайн интерфейса агента (v0.8 Phase 1)

### Что изменилось

**Страница заявки** полностью переработана. Раньше это была длинная страница с карточками информации, которую приходилось прокручивать вниз. Теперь экран разделён на две части:

- **Левая часть** — переписка с клиентом в виде чата. Сообщения клиента отображаются слева, ответы агента — справа. Внутренние заметки (видны только команде) выделены жёлтым фоном с замком. Между сообщениями видны отметки о событиях: «Статус изменён на В работе», «Тикет создан» и т.д. Внизу чата — поле ввода «Напишите ответ...» со кнопкой скрепки для вложений, галочкой «Внутренний» и кнопкой «Отправить».

- **Правая панель** — вся информация о заявке и инструменты управления. Сверху вниз: статус (выпадающий список), прогресс-бар срока реакции (SLA), назначение (кто работает над заявкой), контакт клиента (email, телефон — кликабельны), объект (адрес ЖК/БЦ), классификация, макросы, история событий.

**Вложения** теперь отображаются прямо внутри сообщений — картинки как миниатюры, файлы как карточки с именем и размером. Раньше вложения были в отдельном разделе страницы.

### Новые статусы заявок

Было 5 статусов, стало 7:

| Статус | Что означает | Таймер SLA |
|--------|-------------|-----------|
| Новый | Заявка только поступила | Тикает |
| В работе | Агент работает над проблемой | Тикает |
| Ожидает ответа | Нужна информация от клиента | Остановлен |
| **Отложена** | Заявка приостановлена (ждём оборудование, подрядчика) | Остановлен |
| **Выезд инженера** | Инженер поехал на объект | Тикает |
| Решён | Проблема устранена | Стоп |
| Закрыт | Финальный статус | Стоп |

**Смена статуса** — через выпадающий список в правой панели. Нажимаете на текущий статус, выбираете нужное действие. Система показывает только те переходы, которые доступны из текущего состояния.

**Автоматическая смена статуса:** когда агент отвечает на новую заявку, статус автоматически меняется на «В работе» (раньше сразу переходил в «Ожидает ответа»). Агент сам решает, когда поставить заявку на ожидание.

### Список заявок

- **Вкладка «Открытые»** теперь активна по умолчанию — при входе на портал видны только активные заявки
- **Добавлена вкладка «Выезды»** — показывает заявки со статусом «Выезд инженера»
- **Сортировка «Сначала новые»** — новые заявки наверху

## 2. Утверждение этапов проекта — Approvals (v0.8 Phase 2)

### Как работает

Когда менеджер PASS24 завершает этап проекта внедрения, клиент (администратор УК) получает запрос на подтверждение:

1. Менеджер завершает этап → под карточкой этапа появляется кнопка **«Запросить утверждение»**
2. Клиент получает email: «Этап завершён — подтвердите»
3. На портале клиент видит оранжевый бейдж **«Ожидает подтверждения»** и кнопки **«Утвердить»** / **«Отклонить»**
4. При утверждении — зелёный бейдж **«Утверждено»**
5. При отклонении — клиент пишет причину, этап возвращается в работу, менеджер получает уведомление

## 3. Управление рисками — Risk Tracker (v0.8 Phase 2)

### Как работает

На странице проекта внедрения добавлен таб **«Риски»**. Здесь менеджеры PASS24 ведут реестр рисков:

- **Создание риска:** название, описание, уровень критичности (низкий/средний/высокий/критический), вероятность, воздействие, план снижения
- **Карточки рисков:** цветовая полоска слева по критичности (зелёный → жёлтый → оранжевый → красный), бейдж статуса (Открыт/Снижен/Произошёл/Закрыт)
- **Редактирование:** клик на карточку открывает форму с полями
- **Удаление:** с подтверждением

Только для сотрудников PASS24 (клиент не видит таб «Риски»).

## 4. Редактор шаблонов проектов — Template Editor (v0.8 Phase 2)

### Как работает

В разделе **Настройки → Шаблоны проектов** (только для администраторов) — список всех шаблонов проектов внедрения:

- **Стандартный ЖК** — 10 этапов, ~10 недель
- **Стандартный БЦ** — 9 этапов, ~8 недель
- **Только камеры** — 5 этапов, ~4 недели
- **Большая стройка** — 12 этапов, ~16 недель

Каждый шаблон можно **редактировать** (название, описание, длительность) или **деактивировать** (скрыть из формы создания проекта, но существующие проекты не затронуты).

Шаблоны хранятся в базе данных. При первом обращении автоматически загружаются из встроенных настроек.

## 5. Аналитика проектов — Project Analytics (v0.8 Phase 2)

### Как работает

Страница **/projects/analytics** (доступна через меню «Проекты» → ссылка в интерфейсе) показывает 8 метрик:

| Метрика | Что показывает |
|---------|---------------|
| Всего проектов | Общее количество |
| Активных | Проекты в статусе «В работе» |
| Завершённых | Проекты со статусом «Завершён» |
| На паузе | Проекты со статусом «На паузе» |
| Средняя длительность | Среднее количество дней от начала до завершения |
| Вовремя | Процент проектов, завершённых до плановой даты |
| Открытых рисков | Количество незакрытых рисков во всех проектах |
| Ожидают утверждения | Количество этапов, ожидающих подтверждения клиентом |

Два графика:
- **Проекты по типам** — круговая диаграмма (ЖК, БЦ, Камеры, Большая стройка)
- **Проекты по статусам** — столбчатая диаграмма с цветовой кодировкой

## 6. Gantt-диаграмма (v0.9)

### Как работает

На странице проекта внедрения добавлен таб **«Gantt»** — горизонтальная визуализация всех этапов проекта на шкале времени:

- Каждый этап отображается как полоска, длина соответствует длительности
- Цвет зависит от статуса: серый (ожидает), синий (в работе), зелёный (завершён), красный (заблокирован)
- При наведении — всплывающая подсказка с датами, статусом и процентом прогресса
- Внизу — ползунок масштабирования для навигации по временной шкале

Работает на ECharts (без новых зависимостей).

## 7. WebSocket — обновления в реальном времени (v0.9)

### Как работает

Портал теперь поддерживает подключение через WebSocket для мгновенных обновлений:

- При открытии портала браузер устанавливает WS-соединение с сервером
- Когда кто-то меняет статус заявки или оставляет комментарий — все подключённые агенты получают уведомление мгновенно (без ожидания 15-секундного обновления)
- При обрыве связи — автоматическое переподключение через 5 секунд
- Авторизация через JWT-токен

## 8. Исправления и улучшения

### Email

| Проблема | Решение |
|----------|---------|
| Письма обрезались на 4000 символов | Убрано ограничение, лимит увеличен до 10000 |

| Проблема | Решение |
|----------|---------|
