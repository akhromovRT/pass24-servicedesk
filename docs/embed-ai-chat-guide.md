# AI-помощник PASS24 — Инструкция по встраиванию на любой сайт

## Что это такое

**AI-помощник PASS24** — встраиваемый чат-виджет для сайтов клиентов PASS24. Плавающая кнопка в правом нижнем углу страницы, по клику открывается окно чата с искусственным интеллектом, который:

- Отвечает на вопросы посетителей на основе **базы знаний PASS24** (статьи про приложение, пропуска, доступ, оборудование).
- Если вопрос требует участия человека — предлагает **создать заявку в техподдержку**. Посетитель вводит email, заявка автоматически создаётся в PASS24 Service Desk, агенты поддержки с ней работают, общение продолжается по email.
- Работает **без регистрации и без авторизации**. Любой посетитель сайта может сразу задать вопрос.

**Визуально:** тёмно-синяя шапка с круглым аватаром «AI», подпись «Онлайн», область сообщений с пузырями (ответы AI — серые слева, реплики пользователя — тёмные справа), белый инпут и круглая зелёная кнопка отправки.

Скриншот дизайна: см. ссылку в релиз-заметке от 2026-04-22 (коммит `91bbad7`).

## Быстрый старт (30 секунд)

Вставьте **одну строку** в HTML любой страницы, перед закрывающим `</body>`:

```html
<script src="https://support.pass24pro.ru/chat-loader.js" async></script>
```

Всё. После перезагрузки страницы в правом нижнем углу появится тёмная круглая кнопка с иконкой сообщения. Клик по ней открывает чат.

**Ничего больше не требуется:**
- Не нужен API-ключ.
- Не нужна регистрация домена.
- Не нужно настраивать CORS на вашем сервере.
- Не нужно менять DNS.
- Не нужен HTTPS на вашем сайте (но рекомендуем — иначе браузер может ругаться на mixed content, если ваш сайт по HTTP).

## Как это работает под капотом (архитектура)

```
┌─────────────────────────────────────────┐
│  Сайт клиента (любой домен)             │
│  ┌───────────────────────────────────┐  │
│  │ <script src=".../chat-loader.js">│  │
│  └────────────────┬──────────────────┘  │
│                   │ подгружает          │
│                   ▼                     │
│  ┌───────────────────────────────────┐  │
│  │ Кнопка-пузырь (внизу справа)      │  │
│  │ По клику создаёт iframe           │  │
│  └────────────────┬──────────────────┘  │
│                   │                     │
│  ┌────────────────▼──────────────────┐  │
│  │ <iframe src="support.pass24pro..."│  │
│  │   размер 400×620, прозрачный фон  │  │
│  │ ┌─────────────────────────────┐  │  │
│  │ │ Страница /chat-widget       │  │  │
│  │ │ (Vue-компонент с UI чата)   │  │  │
│  │ │                              │  │  │
│  │ │ → POST /assistant/chat       │  │  │
│  │ │   (AI на базе RAG)           │  │  │
│  │ │ → POST /tickets/guest        │  │  │
│  │ │   (создание заявки)          │  │  │
│  │ └─────────────────────────────┘  │  │
│  └───────────────────────────────────┘  │
└─────────────────────────────────────────┘
                  │
                  ▼
        support.pass24pro.ru
        (наш backend)
```

### Ключевые особенности

1. **Iframe изолирует виджет от сайта клиента.** Наши стили не влияют на ваш дизайн и наоборот. Никаких конфликтов CSS/JS.

2. **Все API-запросы идут из iframe в `support.pass24pro.ru`.** С точки зрения браузера — same-origin (iframe и API на одном домене). Поэтому **CORS не требуется** ни на клиентском сайте, ни на нашем.

3. **Связь «кнопка ↔ iframe»** — через `postMessage`. Например, когда пользователь нажимает «свернуть» внутри чата, iframe отправляет сообщение `{type: 'pass24-chat:collapse'}` родительскому окну, и loader прячет iframe.

4. **AI-логика полностью на нашей стороне.** Клиентскому сайту не нужны ни токены, ни доступ к базе знаний — всё работает через единую точку `support.pass24pro.ru`.

5. **Централизованное обновление.** Если мы обновляем виджет (дизайн, логику, модель AI) — все 250+ сайтов клиентов получают новую версию автоматически при следующей загрузке страницы. Никаких действий на стороне клиентов.

## Параметры кастомизации (опционально)

У тега `<script>` можно задать `data-*`-атрибуты. Сейчас поддерживаются:

| Атрибут | По умолчанию | Описание |
|---|---|---|
| `data-host` | origin из `script.src` | Переопределить домен виджета. Для staging-окружения или собственного развёртывания. |
| `data-z-index` | `2147483000` | z-index кнопки и iframe. Увеличить, если на вашем сайте уже есть overlay с высоким z-index (например, попап согласия на cookies), который перекрывает наш. |
| `data-position` | `bottom-right` | **Начальный** угол виджета: `bottom-right`, `bottom-left`, `top-right`, `top-left`. Используется только при первом визите — после того, как посетитель сам перетащил кнопку, его выбор перебивает этот атрибут (см. ниже «Перетаскивание»). |
| `data-offset-x` | `24` | Отступ кнопки от левого или правого края (px, зависит от `position`). |
| `data-offset-y` | `24` | Отступ кнопки от верхнего или нижнего края (px). Увеличьте, если у вас внизу страницы sticky-панель с кнопками «Сохранить/Отмена». |
| `data-frame-gap` | `76` | Зазор между кнопкой и окном чата по вертикали (px). Обычно менять не нужно. |

Пример — поднять кнопку над sticky-футером с кнопками формы:
```html
<script
  src="https://support.pass24pro.ru/chat-loader.js"
  data-offset-y="80"
  async
></script>
```

Пример — перенести виджет в левый нижний угол (если справа уже стоит Jivo/LiveChat):
```html
<script
  src="https://support.pass24pro.ru/chat-loader.js"
  data-position="bottom-left"
  async
></script>
```

**На старте всем 250+ облачным сайтам достаточно дефолтного варианта без атрибутов.**

## Перетаскивание (только desktop)

Посетитель сайта может **перетащить кнопку** в любой из четырёх углов экрана — это решает ситуацию, когда виджет в стандартном правом нижнем углу мешает что-то нажать (sticky-кнопки формы, элементы виртуального тура, сторонний live-chat). Поведение работает «из коробки», ничего настраивать не нужно.

**Как это выглядит для посетителя:**
- Над кнопкой курсор меняется на «руку» (`grab`).
- Зажав левую кнопку мыши, посетитель тащит виджет в нужную сторону.
- При отпускании кнопка **прилипает к ближайшему углу** (по центру кнопки относительно центра экрана). Свободное позиционирование «куда угодно» сознательно не поддерживается — это исключает ситуацию, когда после ресайза окна или поворота экрана кнопка оказывается за пределами viewport.
- Если в момент перетаскивания чат был открыт — окно тоже перепривязывается к новому углу. **При кнопке вверху экрана окно открывается под ней; внизу — над ней.** Курсор и пространство для текста всегда в направлении центра экрана.

**Запоминание выбора:** позиция сохраняется в `localStorage` под ключом `pass24-chat-corner` для текущего домена. На следующем визите кнопка появится там, где её оставили в прошлый раз. Если у браузера отключён `localStorage` (например, Safari в приватном режиме) — выбор будет сохраняться только до перезагрузки страницы, потом вернётся к `data-position` / дефолту.

**На мобильных (≤480px) drag отключён** — там кнопка остаётся в одном из bottom-углов, а чат разворачивается bottom-sheet'ом на 85% экрана. Разворачивать виджет с верхнего угла на маленьком экране нет смысла.

**Если нужен только конкретный угол без drag** — задайте `data-position` и предупредите посетителей; на drag это не повлияет, но `data-position` определит стартовое положение **до** того, как посетитель что-то перетянул.

## Как работает AI-логика

### Ответы на вопросы

Когда пользователь пишет сообщение:
1. Фронтенд виджета вызывает `POST https://support.pass24pro.ru/assistant/chat` с телом:
   ```json
   {
     "message": "Как сбросить QR-код пропуска?",
     "history": [/* предыдущие сообщения */]
   }
   ```
2. На бэкенде отрабатывает **RAG**:
   - Запрос проходит через **Qdrant** (векторная БД) — ищет релевантные фрагменты статей базы знаний PASS24.
   - Найденные фрагменты + сам вопрос + краткий контекст отправляются в LLM (Claude Sonnet через прокси).
   - LLM формирует ответ **только на основе предоставленного контекста** — если информации в базе знаний нет, так и скажет: «К сожалению, у меня нет ответа — создайте заявку в поддержку».
3. Ответ возвращается с полями:
   ```json
   {
     "reply": "Чтобы сбросить QR-код...",
     "sources": ["slug-статьи-1", "slug-статьи-2"],
     "suggest_ticket": false,
     "ticket_data": null
   }
   ```
4. Виджет показывает ответ пользователю.

### Эскалация в заявку

Есть три триггера, после которых виджет предлагает создать заявку:

1. **AI сам понял, что не знает** (`suggest_ticket: true` в ответе) — обычно при вопросах про конкретную заявку, жалобы, индивидуальные случаи.
2. **Пользователь написал уже 4 сообщения, а AI не помог** — автоматическое предложение.
3. **Ошибка сети** — если API упало, виджет всё равно предложит оставить заявку.

Под ответом AI появляется зелёная кнопка **«Создать заявку в поддержку»**. Клик — показывает inline-форму:
- Email (обязательно)
- Имя (опционально)
- Телефон (опционально)

Заголовок и описание заявки собираются автоматически из истории переписки.

После нажатия «Отправить» виджет вызывает `POST /tickets/guest`, заявка создаётся в PASS24 Service Desk, пользователь получает номер и уведомление: *«Заявка создана — номер XXXXX. Обновления придут на ваш email. Можно отвечать прямо на письма — ответ попадёт в заявку.»*

Дальше агент поддержки работает с заявкой как обычно, ответы приходят клиенту на email.

### Автоматическая привязка заявки к компании-клиенту

Если виджет встроен на сайте формата `https://{name}.pass24online.ru/...` (например, `bristol.pass24online.ru`), заявка автоматически привязывается к компании в реестре постоянных клиентов:

1. `chat-loader.js` подмешивает в URL iframe `?host=bristol.pass24online.ru`.
2. Виджет шлёт это значение как `embed_host` в `POST /tickets/guest`.
3. Backend парсит первый поддомен (`bristol`) и ищет в `customers.subdomain` компанию с этим slug-ом.
4. Если найдена и она помечена как постоянный клиент — в тикете автоматически заполняются `customer_id`, `company` и `object_name`.

В админ-панели Service Desk оператор сразу видит, с какого ЖК пришёл вопрос — без ручной привязки.

Для других доменов (произвольный сайт, тестовая среда) поле просто игнорируется, и заявка создаётся без привязки, как обычно. Чтобы добавить новый объект в реестр, ops-команда заводит запись через seed-скрипт `seed_customer_subdomains.py`.

## Безопасность

### Что отправляется из iframe на наш сервер
- Текст сообщений пользователя в AI.
- Email/имя/телефон, если заполнил форму заявки.

**Мы НЕ собираем автоматически:**
- Cookies вашего сайта (iframe изолирован).
- Данные форм вашего сайта.
- Информацию о посетителях без их явного ввода.

### Защита от zero-shot ошибок
- Виджет не может быть открыт программно внешним JS вашего сайта — это защита от «фантом-кликов» от других виджетов (livechat, аналитика).
- `chat-loader.js` идемпотентен: если его вставить дважды, активным останется только первый экземпляр (`window.__pass24ChatLoaded` флаг).

### CSP / Content Security Policy
Если у вас настроен строгий CSP на сайте, добавьте в него:
```
script-src https://support.pass24pro.ru;
frame-src https://support.pass24pro.ru;
```

## Диагностика проблем

### Кнопка не появляется
1. Откройте **DevTools → Console** — есть ошибки?
2. Откройте **DevTools → Network** → фильтр `chat-loader.js` — загружается ли файл? Какой статус-код?
3. Проверьте, что тег `<script>` добавлен действительно перед `</body>`, а не, например, в `<head>` (тег с `async` отработает и там, но лучше в footer).
4. Проверьте, не блокирует ли AdBlock — редко, но бывает.

### Кнопка есть, но чат не открывается
1. DevTools → Network → посмотрите запрос к `https://support.pass24pro.ru/chat-widget` — 200?
2. Если 500/502/504 — проблема на нашей стороне. Напишите на `support@pass24online.ru`.
3. Если статус OK, но iframe пустой — проверьте CSP: `frame-src 'self' https://support.pass24pro.ru` должно быть разрешено.

### Чат открывается, но AI не отвечает
1. DevTools внутри iframe (правый клик на чате → Inspect) → Network.
2. Ищите `POST /assistant/chat` — какой статус?
   - 401 — проблема с авторизацией (не должно быть для гостя — напишите нам).
   - 429 — вы превысили rate-limit (маловероятно для обычного пользователя).
   - 500 — проблема с AI-моделью / Qdrant на нашей стороне.
3. Если статус 200 но ответ странный — покажите нам `message_id` и `request_id` из response-headers.

### Виджет перекрывается другим элементом
Поднимите z-index:
```html
<script
  src="https://support.pass24pro.ru/chat-loader.js"
  data-z-index="9999999"
  async
></script>
```

### Виджет перекрывает кнопки сайта («Сохранить», «Отмена», «В корзину»)
Самый частый конфликт — у сайта внизу sticky-панель с action-кнопками, и пользователи целятся в «Сохранить», а попадают в AI-чат.

> **Подсказка для пользователя:** конечный посетитель сайта может **сам перетащить кнопку** в любой из четырёх углов (см. раздел «Перетаскивание»). Это доступно на desktop без каких-либо настроек со стороны владельца сайта. Если конфликт мешает только некоторым посетителям — drag решает проблему индивидуально, не трогая глобальные настройки.

Если хочется зафиксировать положение глобально для всех посетителей, есть три способа:

**1) Поднять виджет над панелью (самое простое).** Обычно sticky-футер занимает 56–72px высоты, поэтому `data-offset-y="80"` убирает перекрытие:
```html
<script
  src="https://support.pass24pro.ru/chat-loader.js"
  data-offset-y="80"
  async
></script>
```

**2) Увести виджет в другой угол.** Если правый нижний угол всегда занят — перенесите в левый нижний:
```html
<script
  src="https://support.pass24pro.ru/chat-loader.js"
  data-position="bottom-left"
  async
></script>
```
Также доступны `top-right` и `top-left`.

**3) На уровне сайта:** увеличьте `padding-bottom` у основного контейнера страницы, чтобы контент под sticky-футером не упирался в кнопку виджета. Это полезно, когда кнопки сайта появляются только на некоторых страницах и менять `data-offset-y` глобально не хочется.

### На мобильных устройствах чат некрасиво открывается
Убедитесь, что у вашей страницы есть корректный `<meta name="viewport">`:
```html
<meta name="viewport" content="width=device-width, initial-scale=1">
```
Без него chat-loader не сможет правильно определить мобильный режим.

## FAQ

**Q: Сколько это стоит?**
A: Бесплатно для всех сайтов PASS24 (клиентов облачной системы).

**Q: Нужен ли API-ключ или регистрация?**
A: Нет. Никакой авторизации не требуется — просто вставьте тег.

**Q: Могу ли я менять текст приветствия?**
A: Сейчас — нет (приветствие одно на всех). Если нужна кастомизация — напишите нам, добавим `data-greeting` параметр.

**Q: Что будет, если ваш сервер `support.pass24pro.ru` упадёт?**
A: Кнопка на вашем сайте просто не появится, сам сайт продолжит работать без каких-либо ошибок. Виджет загружается асинхронно (`async`) и не блокирует рендеринг страницы.

**Q: Собирает ли виджет аналитику / cookies?**
A: Только сессионную историю чата (в памяти iframe, очищается при закрытии страницы). Никаких cookies или отслеживания пользователя.

**Q: Работает ли в приватном режиме браузера?**
A: Да. Ничего не сохраняется на диск.

**Q: Поддерживается ли мобильный браузер?**
A: Да. iOS Safari 14+, Chrome Android, Firefox Android. На экранах ≤480px виджет открывается как bottom-sheet на 85% высоты.

**Q: Можно ли встроить одновременно с другими чат-виджетами (LiveChat, Jivo, Tawk)?**
A: Да, они не конфликтуют. Но пользователю может быть путаница с двумя кнопками внизу. Лучше оставить что-то одно, либо разнести их по разным углам через `data-position="bottom-left"` (см. «Параметры кастомизации»).

**Q: Можно ли изменить цвет кнопки или размер окна?**
A: Сейчас — только централизованно на нашей стороне. Если нужно для вашего сайта — напишите, поможем.

**Q: А если я хочу, чтобы AI отвечал только про мой продукт, а не про всю базу знаний PASS24?**
A: Это возможно через `data-context` параметр (заложено на будущее, пока не реализовано). Свяжитесь — приоритизируем.

**Q: Можно ли программно открывать чат (например, при клике на «Связаться с нами»)?**
A: Сейчас — нет, но легко добавить. Скажите — сделаем.

**Q: Как увидеть список заявок, созданных через виджет?**
A: В PASS24 Service Desk → вкладка **Заявки**. Все заявки из виджета помечены `source = web` и имеют заголовок в духе «Вопрос в AI-чате».
