# Инструкция: встраивание AI-чата PASS24 на облачные сайты клиентов

**Назначение.** Поставить плавающую кнопку-помощник AI-чата PASS24 на все ~250 облачных сайтов клиентов. Чат использует нашу базу знаний, может создать заявку в техподдержку от имени гостя. Визуально — тёмно-синяя шапка с AI-аватаром, popup 400×620 в правом нижнем углу.

**Важно:** backend менять не нужно. Все запросы AI и создания заявок идут изнутри iframe в домен `support.pass24pro.ru` (same-origin). На клиентских сайтах не требуется ни CORS-настроек, ни секретов, ни API-ключей.

---

## Что надо сделать

**Добавить одну строку в шаблон сайта — перед закрывающим `</body>`:**

```html
<script src="https://support.pass24pro.ru/chat-loader.js" async></script>
```

Всё. После этого на каждом сайте:

- В правом нижнем углу появится тёмная круглая кнопка-пузырь с иконкой чата.
- По клику открывается popup-чат 400×620 с AI-помощником.
- На мобильных экранах (≤480px) popup раскрывается в bottom-sheet на 85% высоты.

---

## Как это работает под капотом

1. **`chat-loader.js`** — vanilla JS, ~120 строк. Раздаётся статикой с `https://support.pass24pro.ru/chat-loader.js`. Делает:
   - Создаёт кнопку и скрытый iframe в `<body>`.
   - По клику на кнопку — показывает iframe с `src="https://support.pass24pro.ru/chat-widget"`.
   - По клику на «свернуть» внутри iframe (виджет шлёт `postMessage`) — прячет iframe и возвращает иконку чата на кнопке.
   - ESC закрывает открытый чат.
2. **Страница `/chat-widget`** — минимальный Vue-роут на support.pass24pro.ru. Рисует только сам popup (без навбара портала, без всего остального SPA). Общается с API `/assistant/chat` и `/tickets/guest` так же, как существующий AI-чат внутри портала.
3. **Межоконное взаимодействие.** Только `postMessage({ type: 'pass24-chat:collapse' })` из iframe в родителя для сворачивания виджета. Ничего больше.

## Опциональные параметры (если когда-нибудь понадобятся)

У тега `<script>` можно задать `data-*`-атрибуты:

| Атрибут | По умолчанию | Для чего |
|---|---|---|
| `data-host` | origin из `script.src` | Переопределить домен виджета (например, для staging-окружения). |
| `data-z-index` | `2147483000` | z-index кнопки и iframe, если на сайте уже есть свой overlay с высоким z-index. |

Пример:
```html
<script
  src="https://support.pass24pro.ru/chat-loader.js"
  data-z-index="9999"
  async
></script>
```

Пока такие кастомизации не нужны — подойдёт самый короткий вариант из предыдущего раздела.

## Как проверить, что работает

На любом сайте, где вставили loader:

1. Открыть в браузере.
2. Убедиться, что в правом нижнем углу появилась тёмно-синяя круглая кнопка.
3. Кликнуть — должен открыться popup с шапкой «AI-помощник PASS24 • Онлайн» и приветственным сообщением «Здравствуйте! Я AI-помощник PASS24…».
4. Задать любой вопрос (например, «как настроить QR-пропуск»). Через пару секунд придёт ответ от AI на основе нашей базы знаний.
5. Кликнуть на кнопку ещё раз или на «—» в шапке — чат свернётся.

**Консоль браузера.** Не должно быть никаких ошибок CORS / network. Запросы `POST /assistant/chat` видны в Network-вкладке iframe'а и идут на `support.pass24pro.ru`.

## Версионирование и обновления

Центральный deploy: пушим изменения виджета в основной репозиторий `pass24-servicedesk`, GitHub Actions катит на `support.pass24pro.ru`. Все 250 сайтов **сразу** получают новую версию — никаких дополнительных действий на их стороне не нужно (loader всегда подтягивает свежий виджет и свежий JS по прямому URL).

**Кеширование.** Браузеры кешируют `chat-loader.js` и бандл виджета с обычными HTTP-заголовками support.pass24pro.ru. Для жёсткого сброса у конкретного клиента — `Shift+Cmd+R` на их сайте (или `?v=2026-04-21` в URL script'а, но обычно не нужно).

## FAQ

**Q: Можно ли поменять цвет кнопки / позицию?**
A: Сейчас — только через правку `frontend/public/chat-loader.js` в репозитории (централизованно). По запросу добавим `data-color` / `data-position`.

**Q: Нужен ли клиенту логин / куки / домен проверка?**
A: Нет. Виджет работает как гость. Если пользователь захочет создать заявку — введёт email, письмо с номером заявки придёт ему, дальше общение по email.

**Q: Конфликтует ли с другими виджетами (LiveChat, Jivo и т.п.)?**
A: Нет, если у них разные позиции / z-index. Можно поднять z-index нашего через `data-z-index`. В CSS у нашего — префиксы `.pass24-chat-*`, стили изолированы.

**Q: Что будет, если support.pass24pro.ru недоступен?**
A: Кнопка не появится (loader не загрузится), клиент сайта работает как обычно. Ничего не ломается. Ошибки в консоли разработчика — да, но для пользователя невидимо.

**Q: Можно ли встроить виджет в iframe клиента (многослойная вставка)?**
A: Не рекомендуется: наш iframe внутри чужого iframe может попасть под ограничение `X-Frame-Options` / CSP. Для простого embed'а нужен прямой loader-тег на странице.
